# coordination-Post-app
卒業制作 - コーディネート投稿アプリ

---

## サービス概要

本サービスは、オフ会やイベントなど **「人と実際に会う前」** に、  
コーディネートや雰囲気に関する情報を共有することで、  
参加者の心理的な不安を和らげることを目的としたコーディネート投稿アプリです。

主催者がイベントページを作成し、招待URLを共有することで参加者を巻き込める設計としています。  
参加者全員の投稿を前提とせず、一部の参加者や主催者の投稿、  
または簡単な情報共有だけでも当日の雰囲気を想像できるようにすることで、  
イベント参加への心理的ハードルを下げます。

また、イベント開催時以外でも利用できる **自由投稿機能（事前相談用）** を用意し、  
人と会う前の不安を継続的に解消できる場を提供します。

本サービスは **「自分を魅せるためのSNS」ではなく、  
人と会う前の準備・相談のための安心材料を提供することに特化したサービス** です。

---

## このサービスを作った理由

RUNTEQのオフ会に参加した際、参加前は  
「どんな人がいるのか分からない」という不安がありました。

しかし実際に参加してみると、気さくで話しやすい方ばかりで、  
事前の緊張が拍子抜けするほど安心した経験があります。

この体験から、  
**「事前に相手の雰囲気を少しでも知ることができれば、  
人と会うことへの不安を軽減できるのではないか」**  
と考えるようになりました。

私は日常的にファッションから人の個性を感じ取ることが多く、  
ファッションはその人らしさを知るための有効な手段だと感じています。  
この「ファッションを通じて相手を想像できる体験」を形にしたいと考え、  
本サービスを企画しました。

## ユーザー層について

- 10代〜30代
- コーディネートを写真に撮って投稿する習慣がある人
- オフ会やイベントへの参加に興味はあるが、不安を感じやすい人
- 非対面での交流が中心で、実際に会うことにハードルを感じている人

既存のSNSと近い操作体験を提供することで、  
初めて利用するユーザーでも迷わず利用できる設計を想定しています。

## サービスの利用イメージ

### イベント利用（参加前〜当日）

- 主催者がオフ会・イベントページを作成
- 招待URLを共有し、参加者がイベントページに参加
- 参加者は **閲覧のみでも参加可能**
- 主催者、または一部参加者が以下の情報を投稿
  - 簡単なイベントの情報や参加表明など
  - 当日の服装の雰囲気やテイスト(選択式)
  - 持ち物や服装の目印(任意)
  - 当日の服装の写真(任意)

### 自由投稿（イベント以外）

- 次に人と会う予定を想定したコーディネート相談
- 「この服で浮かないか」「無難に見えるか」などの事前確認
- イベント開催の有無に関わらず、準備・相談目的で利用可能

自由投稿においても、  
自己表現を目的とした投稿ではなく、  
**人と会う前の準備・相談** に用途を限定します。

また、自由投稿は「次に人と会う予定を想定したコーディネート相談」を目的としており、
投稿内容の具体性を担保するため、写真投稿を必須とする想定です。

## 投稿ハードルを下げるための工夫

### 初期画面でのメッセージ表示

ログイン前の初期画面において、  
**「本サービスは自己表現SNSではなく、人と会う前の準備・相談のための投稿サービスである」**  
という説明文を **常に表示** する設計としています。

これにより、投稿の目的を明確に伝え、  
完成度の高い投稿を求められているのではないかという誤解を防ぎ、  
投稿に対する心理的ハードルを下げます。

### テンプレート投稿機能

イベント投稿時には、自由入力ではなく **選択式テンプレートを中心** にすることで、  
短時間で投稿が完了する仕組みを想定しています。(自由投稿でも選択可能)

#### 雰囲気（初心者向け・選びやすい）

- やさしい
- 親しみやすい
- きちんと感
- 抜け感
- 無難
- 清潔感

#### テイスト（ファッション好き向け）

- ナチュラル
- ストリート
- フェミニン
- ミニマル
- エッジィ
- ヴィンテージ
- レトロ
- モダン

写真がなくても最低限の情報共有が可能な設計とし、  
入力負荷の軽減を図ります。

## 雰囲気リアクション機能について

本サービスでは、一般的な「いいね」機能の代わりに、
投稿内容に応じたリアクションタグを選択する形式を採用しています。

### 写真あり投稿の場合
コーディネート写真を含む投稿では、
閲覧者が投稿を見て感じた雰囲気を
最大2つまで選択できるリアクションを想定しています。

- やさしい
- 親しみやすい
- きちんと感
- 抜け感
- 無難
- 清潔感

### 写真なし投稿の場合
文章のみの投稿では、
内容を読んで感じた印象や有用性を
最大2つまで選択できるリアクションを想定しています。

- 参考になる
- 分かりやすい
- 安心した
- イメージしやすい
- 助かる
- 無難そう
- 問題なさそう

いずれの場合も数値による評価は行わず、
「どのように受け取られたか」を共有することで、
投稿者が人と会う前に感じている不安の解消につなげることを目的としています。

## ユーザーの獲得について

RUNTEQではオフ会が活発に行われているため、  
オフ会主催者がイベントページを作成し、  
参加者に招待URLを共有することで、自然な形で利用が広がる導線を想定しています。

また、XなどのSNS上で利用事例を共有することで、  
実際の利用シーンを想起してもらい、サービス認知を高めます。

## サービスの差別化ポイント・推しポイント

- イベントページ方式を採用し、主催者主導で参加者を巻き込める
- 参加者全員の投稿を前提としない設計
- イベント以外でも利用できる自由投稿（準備・相談用途）
- 投稿目的を「準備・相談」に限定
- 選択式テンプレートによる入力負荷の軽減
- 数値評価ではなく雰囲気を共有するリアクション設計
- **「人と会う前の不安を和らげる」という明確な利用シーンに特化**

### 競合アプリ調査

#### アプリA（コーディネート検索アプリ）
- 良い点  
  - 検索項目が多く、ユーザー層が幅広い
- 課題点  
  - モデルの体型や実際の雰囲気が伝わりにくい  
  - 交流機能が少ない

#### アプリB（ファッション通販アプリ）
- 良い点  
  - 幅広い商品ラインナップ  
  - 通知による情報提供
- 課題点  
  - PR中心の表示になりやすい  
  - 個人の雰囲気が分かりにくい
